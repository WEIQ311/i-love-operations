# Windows 局域网优化脚本说明

本目录提供 Windows 本机网络优化脚本：

- `lan_network_full_optimize.bat`

该脚本用于对本机网络进行一键排查与优化，重点面向“访问局域网 Web 系统慢、时延高、偶发丢包”等问题。

## 脚本功能

脚本会按顺序执行以下步骤：

1. **管理员权限检查与提权**
2. **优化前基线采集**
   - `ipconfig /all`
   - 网卡状态与速率
   - `netsh wlan show interfaces`
   - 网关 `ping`（若成功识别网关）
3. **基础网络刷新**
   - `ipconfig /flushdns`
   - `ipconfig /renew`
4. **无线电源策略优化**
   - 设置无线网卡为最高性能
5. **Wi-Fi 重连**
   - 断开后尝试按当前 SSID/接口重连
6. **网络栈重置**
   - `netsh winsock reset`
   - `netsh int ip reset`
   - `netsh advfirewall reset`
7. **优化后复测**
   - 重新采集前述关键指标
8. **重启确认**
   - 可选择立即重启（建议）

## 使用方法

### 方法一：双击运行

直接双击：

- `lan_network_full_optimize.bat`

脚本会自动请求管理员权限（UAC 弹窗）。

### 方法二：命令行运行

在 PowerShell 或 CMD 中进入目录后执行：

```bat
lan_network_full_optimize.bat
```

## 日志位置

日志目录：

- `windows/logs/`

日志文件命名格式：

- `lan_optimize_yyyyMMdd_HHmmss.log`

建议每次执行后重点查看：

- `当前接口 / 当前SSID / 默认网关` 是否成功识别
- `[WARN] 命令返回码=...` 是否出现
- 优化前后 `ping` 丢包率和平均时延变化

## 常见现象说明

### 1) 出现“请求的操作需要提升(作为管理员运行)”

说明该步骤需要管理员权限。请确认已同意 UAC 弹窗，或右键“以管理员身份运行”脚本。

### 2) 部分网卡显示“媒体已断开连接”

这是正常现象（如虚拟网卡、蓝牙网卡、未插网线的有线网卡），不一定是故障。

### 3) DNS 显示为非网关地址（如 `100.64.x.x`）

可能存在企业网络策略、VPN、网关接管 DNS 等情况，需结合实际网络策略判断。

### 4) 优化后仍有明显丢包/高延迟

优先排查无线环境问题：

- 路由/AP 距离与遮挡
- 2.4GHz 干扰（建议优先 5GHz/6GHz）
- AP 信道拥堵
- 网卡驱动版本过旧
- 企业 Wi-Fi 认证链路异常

## 安全与变更影响

脚本包含网络栈及防火墙重置操作，可能带来以下影响：

- 某些网络参数恢复默认
- 防火墙规则重置（`netsh advfirewall reset`）
- VPN/代理相关连接可能需要重新配置

如你处于企业受管环境，建议先与网管确认策略再执行。

## 建议的验证标准

执行完成并重启后，建议手动验证：

1. `ping` 默认网关 50 次，观察丢包率与平均时延
2. 打开目标局域网 Web 系统，观察首屏与接口响应
3. 与执行前日志对比链路质量变化

---

如需进一步排障，可结合 `pathping`、`tracert`、DNS 解析耗时做精细定位（本机链路 / DNS / 服务端性能）。
